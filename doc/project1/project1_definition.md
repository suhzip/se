# 프로젝트 I - 프로젝트 정의서

본 프로젝트 정의서는 요구사항 명세서(CRD)를 기반으로 분석 및 재구성하여 작성하였다.

## (1) 프로젝트 명칭 정의

클라우드 파일 공유 시스템 구축 프로젝트


## (2) 개발 시스템 명칭 정의

미니 드라이브 (Mini Drive)


## (3) 개발 시스템에 대한 간략한 설명 (3~5줄)

미니 드라이브는 조직 내에서 발생하는 파일 공유의 비효율성과 보안 문제를 해결하기 위한 클라우드 기반 파일 관리 시스템이다.  
기존 이메일, 메신저, 개인 클라우드를 통한 분산된 파일 관리 방식에서 발생하는 최신 버전 혼선, 검색 어려움, 보안 문제를 해결하는 것을 목표로 한다.  
중앙 집중형 저장 구조를 통해 파일을 통합 관리하고, 권한 기반 공유 기능을 통해 안전하고 효율적인 협업 환경을 제공한다.


## (4) 개발 시스템에서 제공하는 핵심 기능 (5개 이상)

  1. 파일 업로드 및 다운로드 기능  
  2. 폴더 생성, 이동, 삭제 등 파일 관리 기능  
  3. 사용자 간 파일 공유 및 권한 설정 기능  
  4. 파일 이름, 날짜, 업로더 기반 검색 기능  
  5. 파일 버전 관리 및 이전 버전 복원 기능  
  6. 사용자 계정 관리 및 접근 제어 기능  


## (5) 기존 유사 소프트웨어 조사 및 분석

- Google Drive  
  강력한 검색 기능과 실시간 협업 기능을 제공하지만, 데이터가 외부 서버에 저장되어 조직 보안 요구사항과 충돌할 수 있다.
  또한 기업용 요금제 사용 시 지속적인 구독 비용이 발생하여 장기적으로 비용 부담이 존재한다.

- OneDrive  
  Microsoft 서비스와의 연동성이 뛰어나지만 특정 환경에 종속되는 단점이 있다.
  기업 환경에서 사용 시 라이선스 비용이 발생하며, 기능 활용 범위에 따라 추가 비용이 요구될 수 있다.

- Dropbox  
  사용이 간편하고 파일 동기화가 빠르지만, 무료 용량 제한 및 기능 확장에 제약이 있다.
  저장 용량 확장 및 협업 기능 사용을 위해서는 유료 요금제가 필요하여 비용 증가 요인이 된다.

## ✔ 분석 결론

기존 서비스들은 기능적으로 우수하지만, 조직 맞춤형 기능 제공과 보안 통제 측면에서 한계가 존재한다.  
따라서 본 프로젝트는 핵심 기능 중심의 경량화된 사내 전용 시스템을 구축하는 것을 목표로 한다.
