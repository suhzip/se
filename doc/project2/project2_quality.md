# 프로젝트 II - 대상 시스템 품질 요소 추정

요구사항 명세서와 프로젝트 I에서 정의한 시스템 내용을 바탕으로, 미니 드라이브 시스템에서 중요하게 고려해야 할 품질 요소를 이해관계자별로 다음과 같이 정리하였다.

---

## (1) 사용자가 원하는 기대치는 무엇인가?

시스템을 실제로 사용하는 일반 직원이나 외부 협력업체 사용자는 사용 편의성과 빠른 처리 속도를 중요하게 생각한다.

- 안정성  
  파일을 업로드하거나 다운로드하는 과정에서 오류가 발생하지 않고, 시스템이 중단되지 않아야 한다.

- 사용 편의성  
  복잡한 기능 없이 누구나 쉽게 사용할 수 있는 직관적인 화면 구성이 필요하다.

- 처리 속도  
  대용량 파일도 빠르게 업로드 및 다운로드할 수 있어야 하며, 검색 기능 또한 지연 없이 결과를 제공해야 한다.

---

## (2) 개발자 혹은 유지보수 담당자가 원하는 기대치는 무엇인가?

시스템을 개발하고 유지하는 입장에서는 코드 구조와 관리 용이성이 중요하다.

- 코드 이해도  
  코드가 일관된 구조로 작성되어 다른 개발자도 쉽게 이해할 수 있어야 한다.

- 유지보수 용이성  
  이후 기능 추가나 수정이 발생했을 때 기존 시스템에 큰 영향을 주지 않도록 구조가 잘 나뉘어 있어야 한다.

- 테스트 가능성  
  다양한 상황에서 시스템이 제대로 동작하는지 확인할 수 있도록 테스트가 쉽게 이루어져야 한다.

---

## (3) 개발 조직의 관리자(스폰서)가 원하는 기대치는 무엇인가?

프로젝트를 관리하는 입장에서는 비용과 보안, 그리고 안정적인 운영이 중요하다.

- 비용 절감  
  불필요한 기능 개발을 줄이고 핵심 기능 위주로 구현하여 시간과 비용을 효율적으로 사용해야 한다.

- 보안  
  내부 파일이 외부로 유출되지 않도록 접근 권한 관리와 데이터 보호가 중요하다.

- 확장 가능성  
  사용자 증가나 기능 추가에 대응할 수 있도록 유연한 구조로 설계되어야 한다.

---

## ✔ 분석 결론

미니 드라이브 시스템은 사용자에게는 편리하고 빠른 서비스를 제공하면서, 내부적으로는 유지보수가 쉬운 구조를 가져야 한다.  
또한 조직의 입장에서는 보안을 최우선으로 고려하면서 비용 효율적인 시스템을 구축하는 것이 중요하다.
